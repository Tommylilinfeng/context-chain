# CKG Session Summary — 2026-03-18

## What We Built Today

### Cold-start v2 Pipeline (3 Rounds)

Completely rewrote the cold-start pipeline from file-level to function-level analysis with a goal-driven, incremental architecture.

**Round 1 — Scope Selection**
- User provides a natural language goal (e.g. "payment logic", "timeslot system")
- LLM receives all files + function lists from Memgraph CPG
- LLM selects relevant files — no manual path specification needed

**Round 2 — Deep Analysis (per file)**
- Each selected file: full source code + CPG cross-file callers/callees → LLM
- Extracts function-level findings with `related_functions` for multi-anchoring
- Few-shot examples in prompt ensure quality (WHY + trade-offs, not descriptions)

**Round 3 — Synthesis + Classification**
- Receives all Round 2 findings + Business Context from Memgraph
- **Task 1: Classification** — labels each finding as `decision`, `suboptimal`, or `bug`
  - Bugs and suboptimal patterns get a `critique` field explaining the issue
  - Business Context serves as the spec/reference for judging correctness
- **Task 2: Relationship edges** — CAUSED_BY, DEPENDS_ON, CONFLICTS_WITH, CO_DECIDED
- **Task 3: Cross-file multi-anchoring** — decisions relevant to other files get additional ANCHORED_TO edges
- **Task 4: Keyword normalization** — merge synonyms across decisions

### Change Detection & Incremental Analysis

- Git-based change detection between runs (commit-based)
- State tracked in `data/cold-start-state.json` (per-file: lastCommit, decisionIds)
- Unchanged files skipped; old decisions deleted before re-analysis
- **Default mode**: re-analyze if file changed OR any callee file changed
- **Deep check mode** (`--deep-check`): also re-analyze if any caller file changed
- **Force mode** (`--force`): ignore all change detection

### Dashboard (Unified UI)

- Shared sidebar navigation + consistent dark theme (DM Sans + JetBrains Mono)
- **System page**: Memgraph status, config validation, CPG status, one-click setup actions
- **Cold Start page**: goal input, SSE log streaming, round progress, force/deep-check/dry-run options
- **Decisions page**: bug/suboptimal badges with critique blocks
- All pages (Overview, Coverage, Graph, Business Context) restyled

## Known Issues (Not Yet Fixed)

### P0: index.ts collision
biteme-infra's 5 edge functions all named `index.ts`. Anchoring by `name` causes cross-wiring. Fix: use file `path` or full node ID.

### P1: Round 3 only sees summary, not content
`DecisionSummary` lacks `content` field. Bug detection quality limited. Fix: include content in Round 3 prompt.

### P1: Business Context not in Round 2
Round 2 analyzes code without business expectations. Fix: pass Business Context to Round 2 prompt.

### P2: Cross-repo isolation
CPG queries scoped to single repo. bite-me-website to biteme-shared calls invisible. Fix: query CALLS_CROSS_REPO edges.

### P2: related_functions anchoring scoped to same repo
Fix: drop repo constraint or try all repos.

### P3: Joern not automated
Run manually before pipeline. Could add one-click Joern + Ingest + Pipeline.

### P3: Session ingestion still v1
`ingest-sessions.ts` uses old file-level logic. Could benefit from CPG-driven function-level anchoring.
